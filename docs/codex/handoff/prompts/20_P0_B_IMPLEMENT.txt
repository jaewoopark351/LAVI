# P0-B implementation — Core runtime, config, smoke

P0-A review가 승인된 상태에서 P0-B만 구현하라.

먼저 공통 계약과 P0 implementation/review 문서를 읽어라.

이미 존재하는 다음 구현을 중복 생성하지 마라.

- core/profile_resolver.py
- config/modules.core.json
- plugins/NullInput
- plugins/NullLLM
- plugins/NullTTS
- plugins/NullVtuber

범위:

- 기존 resolver/Null provider acceptance 검증과 최소 보완
- 일반 production config와 Core config resolution 계약
- core/paths 기반 config path 및 안전한 사용자 override write
- Core startup의 FFmpeg/network/model/process unconditional side effect 차단
- scripts 기반 Core offline smoke
- tracked audit의 production configuration smoke와 --production-config-smoke 유지
- attempt counters, timeout, shutdown

금지:

- root modules.json 변경/migration/rewrite
- example config production fallback
- P1 공통 lifecycle/manifest 전면 구현
- python -m lavi를 위해 package 신설
- commit/push

작업 전후 modules.json hash를 기록하고, Core smoke와 production config smoke 결과를 보고한 뒤 멈춰라.
